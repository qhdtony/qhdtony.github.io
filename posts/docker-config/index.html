    Docker Config

    August 18, 2025

    今天有个老项目要用到PHP7.4 环境，官方已经不建议了。 做一个笔记总结一下遇到的坑

    零、文件结构大概这样：#
    lnmp-docker/
├── docker-compose.yml
├── nginx/
│   └── default.conf
├── php/
│   └── Dockerfile
└── src/
    └── index.php
    一、版本过低的问题#

    一直报错的是 xdebug 装不上，我就抛弃了 xdebug

    环境比调试用得更多。

    二、镜像源的问题#
    设置-docker engine

{
  "builder": {
    "gc": {
      "defaultKeepStorage": "20GB",
      "enabled": true
    }
  },
  "experimental": false,
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://hub-mirror.c.163.com",
    "https://mirror.baidubce.com"
  ]
}
    三、服务内调用的问题-redis#
    
Fatal error: Uncaught RedisException: Connection refused in index.php: Stack trace: #0 Redis->connect('127.0.0.1', 6379) #1 {main} thrown in /var/www/html/index.php on line 5` 

代码里把 127.0.0.1 写成 redis, 服务名称叫 lnmp-redis
    四、代码同步的问题#

    本地设置的目录是 src volumes 映射的目录是 /var/www/html 一般改了src的文件， docker里面的映射目录会立即更新

    nginx:
    image: nginx:1.25
    container_name: lnmp-nginx
    ports:
      - "8080:80"
    volumes:
      - ./src:/var/www/html:cached
      - ./nginx/default.conf:/etc/nginx/conf.d/default.conf:ro
    depends_on:
      - php
    五、进入命令行的问题#
    进入容器

docker exec -it lnmp-php bash
    六、其他命令#
    docker ps #查看容器服务状态
docker exec -it lnmp-php sh #容器没有bash
docker run -it php:7.4-fpm bash #启动时直接进入

-i：交互模式
-t：分配一个伪终端
exec：在容器里执行命令

启动 LNMP 环境
打开 Docker Desktop，确保 Docker 正在运行。
打开终端进入项目目录：
cd ~/lnmp-docker
docker compose up -d --build
--build 确保 PHP 镜像按照新 Dockerfile 重建。
浏览器访问：
PHP 页面：http://localhost:8080
phpMyAdmin：http://localhost:8081
